# GAS学習支援Webアプリ 要件定義書

## 1. プロジェクト概要

### 1.1 目的
「AIと学ぶ実践プログラミング講座」のGAS学習フェーズを支援するWebアプリケーション。学習コンテンツの配信、受講生の進捗管理、課題提出・フィードバックの機能を提供する。

### 1.2 関連事業
生成AI活用プログラミング学習支援事業（2025年4月開講予定）

### 1.3 技術スタック
| 項目 | 技術 |
|:--|:--|
| フロントエンド | Next.js 16.0.10 (App Router) |
| UI フレームワーク | React 19.2.1 |
| スタイリング | Tailwind CSS 4 |
| 言語 | TypeScript 5 |
| ランタイム | bun 1.2.21 |
| コード品質 | Biome 2.3.9 (リンター + フォーマッター) |
| バックエンド / DB / 認証 | Supabase |
| ホスティング | Vercel |
| 運用コスト | 無料枠で運用（初期） |

## 3. 機能要件

### 3.1 認証・ユーザー管理

#### 3.1.1 ログイン
- Googleアカウントによるログイン（Supabase Auth）
- 他の認証方法は提供しない

#### 3.1.2 新規登録・承認フロー
1. 受講生がGoogleアカウントで登録申請
2. 登録時に入力する情報：
   - 名前
   - メールアドレス（Googleアカウントから自動取得）
3. 運営側にSlack通知
4. 運営側が管理画面で「承認」または「却下」
5. 承認後、受講生にメール通知
6. 承認された受講生のみコンテンツにアクセス可能

#### 3.1.3 未承認状態
- ログイン後は「承認待ち」画面のみ表示
- コンテンツへのアクセス不可

---

### 3.2 学習コンテンツ配信

#### 3.2.1 コンテンツ構成
```
Phase（例：Phase 1 - GAS基礎）
  └── Week（例：Week 1 - はじめの一歩）
        ├── 動画教材（複数）
        ├── テキスト教材（複数）
        └── 演習課題（複数）
```

#### 3.2.2 動画教材
- YouTubeの限定公開動画を埋め込み表示
- 運営側でYouTube URLを登録

#### 3.2.3 テキスト教材
- Webアプリ内にリッチテキスト / Markdown形式で表示
- 運営側で直接編集・管理

#### 3.2.4 演習課題
- 課題の説明文を表示
- 提出方法の案内
- 提出期限の目安（強制ではない）

---

### 3.3 進捗管理

#### 3.3.1 進捗の記録方法
- 各コンテンツ（動画・テキスト・演習）に「完了」ボタン
- 受講生が手動でクリックして完了を記録

#### 3.3.2 受講生側の進捗表示
- 全体の進捗率（%表示）
- Phase単位の完了状況
- Week単位の完了状況
- 各コンテンツ単位の完了チェック

#### 3.3.3 講師側の進捗確認
- 受講生一覧 + 各受講生の進捗率
- 個別受講生の詳細進捗（どのコンテンツまで完了か）

---

### 3.4 課題提出・フィードバック

#### 3.4.1 提出方法
受講生は以下のいずれかの形式で提出：
- GASコードをテキストで貼り付け
- スプレッドシート / GASプロジェクトのURL共有

#### 3.4.2 提出フロー
1. 受講生が課題を提出
2. 運営側にSlack通知
3. 講師が管理画面で内容を確認
4. 講師がコメント / フィードバックを記入
5. 受講生にメール通知 + アプリ内通知
6. 受講生がフィードバックを確認

#### 3.4.3 補足
- 点数・合否判定は行わない
- 提出期限は目安のみ（強制なし）

---

### 3.5 通知機能

#### 3.5.1 運営側への通知（Slack）
- 新規登録申請があった時
- 課題が提出された時

#### 3.5.2 受講生への通知（メール + アプリ内）
- アカウント承認完了時
- 課題にフィードバックが返信された時
- お知らせが投稿された時（任意）

---

### 3.6 お知らせ機能

#### 3.6.1 概要
運営から全受講生への連絡機能

#### 3.6.2 表示場所
- ホーム画面（ログイン後のトップ）
- 専用のお知らせ一覧ページ

#### 3.6.3 お知らせの内容例
- メンテナンス情報
- 新コンテンツ追加のお知らせ
- イベント・オフィスアワーの案内

## 5. 非機能要件

### 5.1 対応デバイス
- PC：メイン利用を想定、フル機能
- スマートフォン / タブレット：レスポンシブ対応、情報確認用途

### 5.2 パフォーマンス
- 初期受講生規模：10〜20名
- Supabase / Vercel無料枠で対応可能な範囲

### 5.3 セキュリティ
- Supabase Authによる認証
- 承認済みユーザーのみコンテンツアクセス可
- Row Level Security（RLS）でデータアクセス制御
